# 示例文档模板规范

## addition.md 文档结构要求

每个示例程序（LD和CFC）的 `addition.md` 文档必须包含以下章节：

### 1. 概述（Overview）
- 示例程序的简要说明
- 实现的功能（加法运算）
- 适用的编程语言类型（LD或CFC）

### 2. 变量说明（Variables）
必须包含完整的变量表格：

| 变量名 | 数据类型 | 方向 | 说明 |
|--------|----------|------|------|
| A | INT | INPUT | 加数1，范围-32768到32767 |
| B | INT | INPUT | 加数2，范围-32768到32767 |
| Result | INT | OUTPUT | 和，可能发生溢出回绕 |

### 3. 程序逻辑说明（Logic Description）
- 功能块说明（ADD功能块）
- 数据流向图或文字描述
- 逻辑连接关系

### 4. 导入步骤（Import Instructions）
详细的导入指南，包括：
1. 打开目标开发工具（CODESYS/TIA Portal/CX-Programmer）
2. 创建新项目或打开现有项目
3. 导入 `addition.xml` 文件的具体步骤
4. 验证导入成功的方法

### 5. 运行指南（Execution Guide）
1. 如何设置输入变量（A和B）的值
2. 如何启动程序执行
3. 如何查看输出结果（Result）
4. 如何修改输入值并重新运行

### 6. 测试用例（Test Cases）
必须包含所有5个测试场景：

| 测试场景 | A值 | B值 | 期望Result | 说明 |
|----------|-----|-----|-----------|------|
| 正常加法 | 10 | 20 | 30 | 基本功能验证 |
| 负数加法 | -5 | 3 | -2 | 负数处理 |
| 正向溢出 | 32767 | 1 | -32768 | 溢出回绕行为 |
| 负向溢出 | -32768 | -1 | 32767 | 负向溢出回绕 |
| 零值 | 0 | 0 | 0 | 边界值测试 |

### 7. 注意事项（Notes）
- INT数据类型的范围限制
- 溢出回绕行为说明
- 工具兼容性提示
- 常见问题解答（如有）

### 8. CFC特定章节（仅CFC示例）
如果是CFC示例，必须额外包含：
- **LD与CFC对比**: 说明两种编程方式的关键差异
- **功能块图特点**: CFC数据流可视化的优势

---

## 文档质量要求

- **语言**: 100%使用简体中文
- **术语一致性**: 使用标准IEC61131-3术语
- **可读性**: 面向初学者，避免复杂技术术语
- **完整性**: 覆盖所有必需章节
- **准确性**: 所有步骤经过实际工具验证
